# CLAUDE.md

> Read this at the start of every session.

## Project

**Forge** — versioned Claude skills for structured agentic software development.

This repo IS the skills. Work here means adding, improving, or reviewing skill files
in `skills/`. Each skill is a Claude plugin installable from the marketplace.

---

## Session start

1. Read this file
2. Review `skills/` for what needs work
3. Run `/skill-creator` to create or improve a skill

---

## Repo structure

```
skills/<name>/         ← skill source (SKILL.md + bundled resources)
dist/plugins/<name>/   ← compiled plugin packages (generated by scripts/build-dist.js)
.claude-plugin/        ← marketplace manifest
scripts/               ← repo-level scripts
```

After modifying a skill:

1. Regenerate dist/:
```bash
node scripts/build-dist.js
```

2. Bump the version in `.claude-plugin/marketplace.json` and add an entry to `CHANGELOG.md`:
   - `fix` or internal change → patch (`x.x.N`)
   - new skill behaviour or capability → minor (`x.N.0`)
   - breaking change → major (`N.0.0`)

---

## Commit convention

```
type: short imperative description
```

Types: `feat`, `fix`, `docs`, `refactor`, `chore`

Examples:
```
feat: add open decision mode to write-adr
fix: add guard for missing plan.md in implement
docs: update code-reviewer template
```

---

## Principles

- Each skill must be self-contained — no references to private repos or personal paths
- SKILL.md stays under 500 lines — heavy content lives in bundled resources
- Every change to a skill must update the corresponding dist/ entry
- Skills are portfolio signals: they show structured, intentional agentic development
