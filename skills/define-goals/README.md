# define-goals

Creates or evolves `GOALS.md` — the product source of truth for a project. Goals describe outcomes for specific people, never solutions.

## When to use

Say "define goals", "update goals", "review our goals", or "what are we building toward". Also triggered automatically at the end of `start-project`.

## Two modes

- **Inception** — no `GOALS.md` exists yet. Creates it from scratch, seeded by README and project context. Asks for the demo-ready threshold (Milestone 1 = MVP).
- **Evolution** — `GOALS.md` already exists. Reviews current goals, proposes what to add, extend, or close out.

## Output

`GOALS.md` at the repo root — linked from README, drives issue creation.
